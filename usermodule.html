👤 App User Module

The App User module manages user registrations, credentials, and verification privileges to ensure scalable, secure workspace access.

About App User

An App User represents every individual who accesses the Virtual Sticky Note Brainstorming Platform. Users can register, log in, create brainstorming boards, write sticky notes, invite team members, and collaborate with others. Every board, sticky note, member, and activity is directly linked to an App User account, making this module the foundational layer of our relational database architecture.

User Roles & Access Control

The platform enforces role-based access governance to preserve workspace separation and data integrity:

🛠 Facilitator

Retains complete administrative privileges over the workspace. Responsible for executing all operations, including creation, modification, configuration, and final deletion of canvas layers.

✏ Contributor

An active workspace participant provisioned with extensive creation and modification rights. Can fully edit sticky notes and layouts, but is explicitly restricted from deleting data structures.

👁 Stakeholder

A read-only reviewer with strict viewing permissions. Restricted to querying platform resources via high-level endpoints (GetAll) or locating precise system entities (GetById).

Module Objectives

🎯 1. Secure Onboarding

Provide validation during registration workflows to ensure data authenticity and system resource reservations.

🎯 2. Seamless Team Alignment

Support dynamic canvas tracking to enable active contributor attribute mapping across shared sessions.

🎯 3. Access Protection

Isolate sensitive organizational structures from unauthorized entry through programmatic session tokens.

🎯 4. High Concurrency

Deliver ultra-low latency response cycles during continuous updates from concurrent user inputs.

🎯 5. Accountability Logging

Generate clear history tracking entries for user events to audit past modifications easily.

Privacy & Authentication Security

Protecting identity details within the App User Module remains paramount. The infrastructure integrates the following security controls sequentially:

  • Secure Token Authentication (JWT): Manages active user sessions seamlessly across dynamic tabs to avoid credentials hijacking.
  • Cryptographic Password Hashing: All sensitive baseline user passwords undergo irreversible mathematical hashing prior to storage in database layers.
  • Session Expiration Logs: Suspends passive client interactions automatically after prolonged periods to secure open public terminals.
  • Profile Autonomy Controls: Offers users direct options to download account historical telemetry datasets or command full deletion audits.
